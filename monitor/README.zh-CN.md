# Comet Invitation Hunter – 后台监控服务

该目录包含一个后台监控服务，它会持续在 X（Twitter）上搜索与 Comet 浏览器邀请相关的帖子，并向已验证的用户发送邮件通知。

## 概述

该监控服务的工作流程如下：

1. **搜索**：同时使用多个与 Comet 相关的关键词在 X 上进行搜索  
2. **去重**：去除不同关键词搜索结果中的重复帖子  
3. **分类**：将帖子区分为“免费分享”型（含直接链接）或“条件分享”型（需满足特定条件方可获取）  
4. **存储**：将新发现的邀请帖子保存至数据库  
5. **通知**：向所有已验证用户批量发送邮件通知  

## 组件说明

### PostClassifier
- 负责将帖子分类为“免费分享”或“条件分享”
- 从“免费分享”类帖子中提取邀请链接
- 识别“条件分享”类帖子中所需的操作步骤

### EmailNotifier
- 生成 HTML 格式及纯文本格式的邮件通知
- 采用指数退避算法实现重试机制
- 记录每一次邮件发送尝试的结果

### CometMonitor
- 统筹整个监控流程的执行
- 具备错误恢复能力并可持续运行
- 管理 API 调用频率限制以及数据库连接

该服务依赖以下环境变量进行配置：


## 搜索关键词

该服务会搜索包含以下关键词的帖子：
- `perplexity.ai/browser/claim`（直接提供链接）
- `comet invitation`
- `comet invite`
- `comet browser invite`
- `comet access`
- `perplexity browser invite`
- `ai browser invite`

## 帖子分类规则

### 免费分享类帖子
- 包含直接的 Comet 邀请链接：`https://www.perplexity.ai/browser/claim/[CODE]`
- 用户可直接使用该链接获取邀请资格
- 此类帖子在存储时会一并保存提取出的邀请链接

### 条件分享类帖子
- 包含诸如“私信我”、“关注后私信”或“在下方留言”等表述
- 同时必须出现与 Comet 相关的关键词
- 存储时还会记录对应的操作条件与要求

## 邮件通知机制

- 每当发现新帖子后，系统即发送通知邮件
- 单封邮件中会同时包含“免费分享”与“条件分享”两类帖子信息
- 邮件采用 HTML 格式，内含帖子链接及邀请链接
- 重试机制可应对临时性的邮件发送失败情况

## 错误处理策略

- **API 频率限制**：通过在请求间加入延迟来实现平滑处理  
- **数据库故障**：使用指数退避算法进行重试  
- **邮件发送失败**：每个批次最多尝试重试 3 次  
- **持续运行**：即便出现个别错误，服务仍会继续运转  

## 日志记录

所有操作均会以相应级别进行记录：
- INFO：常规操作，如发现新帖子或成功发送通知  
- WARNING：频率限制触发、重试发生或配置缺失等情况  
- ERROR：操作失败、API 报错或数据库异常  
- DEBUG：分类逻辑细节及 API 返回内容

## 数据库结构

该服务使用以下数据表：

- `posts`：存储经处理的邀请帖子信息  
- `users`：保存已验证用户的电子邮箱地址  
- `email_logs`：记录通知批次及其发送状态
